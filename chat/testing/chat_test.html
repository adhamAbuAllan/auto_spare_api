2. Conversation Setup And REST Snapshots

Use these buttons for the REST endpoints that were missing from the old browser tester.

Recommended flow: create or log in both users, create the conversation as User A, add User B as a participant, then confirm the conversation and message history with the GET buttons before opening websocket chat.
Typing and presence are websocket runtime signals. They will not show up in GET /api/messages/ or GET /api/conversations/. Message creation and seen status will.
